# 三角洲行动 PC 地图标点工具设计

## 目标

在当前目录创建一个面向三角洲行动 PC 端的工具集，第一版实现地图标点工具。工具以外部桌面覆盖层方式运行，用于在游戏地图上显示物资点位，并通过配置文件适应游戏版本变动。

第一版目标：

- Windows 桌面工具。
- 使用 Tauri + TypeScript 实现。
- 不注入游戏进程，不修改游戏文件，不读写游戏内存。
- 在游戏内按 M 呼出地图后，覆盖层显示当前场景地图的物资点位。
- 支持多张场景地图。
- 支持不同物资类型使用不同 logo，例如钻石点显示钻石图标，房卡点显示房卡图标。
- 支持点击点位显示详情和可选实景截图。
- 支持浏览、交互、编辑三种模式。
- 配置组织清晰，方便游戏版本变动后快速增删改点位和物资类型。

## 非目标

第一版不包含：

- 自动识别当前游戏地图。
- 自动识别游戏内地图区域。
- 自动导入或整理截图文件。
- 云端同步。
- 游戏进程注入、游戏内存读取、游戏文件修改或反作弊绕过。
- 复杂搜索、复杂标签组合筛选。

## 技术方案

采用 Tauri + TypeScript：

- 前端：TypeScript UI，负责覆盖层渲染、设置窗口、点位详情、编辑表单和筛选面板。
- Tauri/Rust 后端：负责透明置顶窗口、点击穿透切换、全局热键、配置读写和用户设置持久化。
- 配置：使用 JSON 文件组织物资类型、地图点位和用户设置。

选择该方案的原因：

- 比 Electron 更轻量，适合后续发布给普通用户。
- UI 使用 Web 技术，方便后续扩展点位编辑器和配置管理界面。
- Tauri 后端适合处理 Windows 桌面窗口能力和文件系统能力。

## 窗口架构

### 覆盖层窗口

覆盖层窗口用于叠加在游戏画面上：

- 全屏。
- 透明。
- 置顶。
- 无边框。
- 根据模式切换点击穿透状态。
- 根据当前地图和筛选状态渲染点位 logo。

坐标采用相对坐标，点位配置中的 `x` 和 `y` 范围为 `0~1`：

```text
screenX = x * screenWidth
screenY = y * screenHeight
```

第一版默认假设用户全屏显示游戏地图。后续预留手动校准能力，每张地图可保存：

```json
{
  "offsetX": 0,
  "offsetY": 0,
  "scale": 1
}
```

### 设置/控制窗口

设置窗口是普通桌面窗口，用于：

- 选择当前地图。
- 显示和切换当前模式。
- 按物资类型显示/隐藏点位。
- 显示当前覆盖层状态。
- 显示快捷键说明。
- 在编辑模式下提示是否有未保存更改。
- 保存当前地图配置。

## 模式设计

### 浏览模式

默认模式：

- 覆盖层点击穿透。
- 不影响游戏鼠标操作。
- 只显示当前地图、当前筛选条件下的点位 logo。
- 适合玩家打开游戏地图时快速查看物资点位。

### 交互模式

用于查看点位信息：

- 覆盖层取消点击穿透。
- 点击点位 logo 显示详情浮窗。
- 不修改配置。

点位详情包含：

- 标题。
- 物资类型。
- 描述。
- 标签。
- 截图区域。

截图逻辑：

- `screenshots` 为空时显示“暂无实景截图”。
- 有截图时显示第一张。
- 多张截图时支持上一张/下一张。
- 图片路径不存在时显示“截图缺失”，但应用不崩溃。

### 编辑模式

第一版需要编辑模式，用于维护点位配置：

- 覆盖层取消点击穿透。
- 在覆盖层空白处点击新增点位。
- 点击已有点位打开编辑表单。
- 拖拽已有点位修改位置。
- 支持删除点位。
- 修改先保存在内存状态中，显式保存后写回当前地图 JSON。

新增点位流程：

1. 用户选择当前地图。
2. 切换到编辑模式。
3. 用户在覆盖层上点击目标位置。
4. 工具把点击位置换算为相对坐标。
5. 弹出新增点位表单。
6. 用户选择物资类型，填写标题、描述、标签，可选填写截图路径。
7. 保存到内存状态，标记当前地图有未保存更改。
8. 用户点击“保存当前地图配置”后写回 JSON。

点位表单字段：

- ID：自动生成，可手动修改。
- 物资类型：下拉选择。
- 标题：文本。
- 描述：文本。
- 标签：逗号分隔。
- 截图路径：可选，可填写多个相对路径；默认空数组。
- 坐标：根据点击或拖拽自动生成，也允许手动微调。

保存策略：

- 使用显式保存。
- 切换地图或退出前，如有未保存更改则提示。
- 保存前校验数据结构。
- 保存前备份原地图文件，例如 `config/maps/zero-dam.json.bak`。
- 保存失败时保留内存改动，不覆盖原文件。

## 快捷键

建议第一版快捷键：

- `Ctrl + PageUp`：切换上一张地图。
- `Ctrl + PageDown`：切换下一张地图。
- `Ctrl + Alt + Space`：切换浏览模式和交互模式。
- `Ctrl + Alt + E`：切换编辑模式。
- `Ctrl + Alt + H`：显示或隐藏覆盖层。

`M` 是游戏内地图呼出按键。第一版不主动接管 `M`，避免和游戏按键冲突。

## 配置结构

配置分为版本配置和用户设置。

```text
config/
  loot-types.json
  maps/
    zero-dam.json
    longbow-valley.json

user-data/
  settings.json

assets/
  icons/
    diamond.svg
    keycard.svg
    safe.svg

  screenshots/
    zero-dam/
      diamond-001-1.jpg
      diamond-001-2.jpg
      keycard-003-1.jpg
```

### 物资类型配置

`config/loot-types.json`：

```json
{
  "version": 1,
  "types": [
    {
      "id": "diamond",
      "name": "钻石",
      "icon": "assets/icons/diamond.svg",
      "color": "#40D9FF",
      "defaultSize": 28,
      "valueLevel": 5,
      "defaultVisible": true
    },
    {
      "id": "keycard",
      "name": "房卡",
      "icon": "assets/icons/keycard.svg",
      "color": "#FFCC33",
      "defaultSize": 28,
      "valueLevel": 5,
      "defaultVisible": true
    }
  ]
}
```

物资点位只引用 `type`，具体 logo、显示名称、颜色和默认尺寸由物资类型配置决定。

### 地图点位配置

`config/maps/zero-dam.json`：

```json
{
  "version": 1,
  "id": "zero-dam",
  "name": "零号大坝",
  "defaultCalibration": {
    "offsetX": 0,
    "offsetY": 0,
    "scale": 1
  },
  "points": [
    {
      "id": "diamond-001",
      "type": "diamond",
      "x": 0.532,
      "y": 0.418,
      "title": "二楼保险旁",
      "description": "靠近保险箱刷新点，注意窗口方向。",
      "screenshots": [
        "assets/screenshots/zero-dam/diamond-001-1.jpg"
      ],
      "tags": ["高价值", "室内"]
    },
    {
      "id": "keycard-003",
      "type": "keycard",
      "x": 0.274,
      "y": 0.661,
      "title": "宿舍桌面",
      "description": "",
      "screenshots": [],
      "tags": ["房卡"]
    }
  ]
}
```

每张地图独立一个 JSON，方便游戏改版后单独替换。

### 用户设置

`user-data/settings.json`：

```json
{
  "selectedMapId": "zero-dam",
  "visibleLootTypes": {
    "diamond": true,
    "keycard": true,
    "safe": false
  }
}
```

第一版只保存：

- 当前选择的地图。
- 各物资类型是否显示。

用户设置不和点位配置混在一起，避免更新配置包时覆盖用户习惯。

## 筛选设计

第一版支持按物资类型筛选：

- 设置窗口显示物资类型复选框。
- 支持显示或隐藏某类物资点。
- 筛选状态保存到用户设置。
- 新增未知类型时，默认使用物资类型配置中的 `defaultVisible`。

复杂筛选如价值等级、标签组合、危险等级等只预留字段，第一版不实现复杂筛选 UI。

## 地图切换

支持两种方式：

- 设置窗口下拉选择当前地图。
- 快捷键循环切换地图。

切换地图后：

- 覆盖层立即重新渲染点位。
- 设置窗口同步当前地图。
- 保存 `selectedMapId`。
- 如当前地图有未保存编辑，切换前提示保存或放弃。

## 错误处理

启动和运行时需要保证配置错误不会导致应用崩溃。

加载配置：

- `loot-types.json` 读取失败：显示错误页，提示检查配置。
- 某张地图 JSON 读取失败：跳过该地图，并在设置窗口提示。
- 点位 `type` 不存在：使用默认图标显示，并标记为未知类型。
- 点位坐标不在 `0~1`：跳过该点位，或在编辑模式提示修复。
- icon 路径不存在：使用默认占位图标。
- screenshot 路径不存在：详情浮窗显示“截图缺失”。

保存配置：

- 保存前校验地图数据结构。
- 保存前创建 `.bak` 备份。
- 保存失败时不覆盖原文件。
- 保存失败后保留内存改动，允许用户重试。

## 测试范围

第一版测试重点：

- 配置解析：正常加载物资类型、多张地图、未知物资类型、坐标越界。
- 用户设置：保存当前地图和物资类型筛选状态。
- 覆盖层渲染：根据当前地图和筛选状态显示点位，根据相对坐标计算屏幕位置。
- 模式切换：浏览模式点击穿透，交互模式可点击，编辑模式可新增、拖动、编辑、删除。
- 点位详情：有截图、无截图、截图缺失、多截图切换。
- 编辑保存：新增点位写入当前地图 JSON，修改后显式保存，保存前生成备份，未保存改动切换地图时提示。

## 发布与风险提示

发布给普通用户时，应说明：

- 工具是外部覆盖层，不注入游戏进程，不修改游戏文件。
- 任何游戏覆盖层工具都可能被游戏或反作弊系统视为风险行为。
- 用户应自行确认使用风险。
